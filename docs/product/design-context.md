# Claread 设计上下文

本文描述 Claread 当前可继承的产品气质和设计约束。它不是旧设计过程记录，也不包含微信小程序专属实现细节。

## 产品气质

Claread 的体验关键词是：

- Calm：阅读过程安静，不用炫技动效打断理解。
- Precise：解释、标注、例句和词典结果应尽量精确。
- Editorial：界面应像经过编辑整理的阅读材料，而不是工具堆叠。

## 产品层级

优先级从高到低：

1. 阅读理解。
2. 语境化解释和标注。
3. 生词、批注、反馈等用户资产。
4. 英语学习工具能力。

后续 Web 端可以提升表现力，但不能牺牲阅读主线。

## 设计原则

- 先展示文章和解释，再展示工具。
- 复杂能力渐进披露，不在第一屏暴露所有操作。
- 反馈、收藏、生词、批注都应围绕阅读上下文出现。
- 多端共享业务契约，但不强行共享 UI 实现。
- 小程序端可以降级，Web 端应追求更完整的阅读体验。

## 反模式

- 把 Claread 做成通用 AI chat 工具。
- 把阅读页做成按钮和卡片堆叠。
- 让用户先配置复杂选项再开始阅读。
- 用平台限制污染后端 canonical result。
- 把小程序端的降级 UI 当作 Web 端上限。

## 多端说明

当前第一个稳定客户端是微信小程序。新仓库中，小程序只是 `apps/miniprogram/` 客户端之一。

Web、APP、Directus、eval/RAG 后续都应围绕同一套后端数据和业务契约演进。
