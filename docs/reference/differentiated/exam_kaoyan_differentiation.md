## 1. 文档定位

本文档结合考研英语阅读的考察特征与 Claread 的输出 Schema，分析在 `kaoyan` variant 下各组件的解析策略、密度控制、重点关注事项，并提供可直接用于 runtime prompt 注入的 `scenario_policy` 建议。

`kaoyan` 是一个统一的 variant，不区分英语一和英语二。本文档提取两者的共性作为策略基础，对差异部分取**偏英语一的上限**（因为能处理英语一的策略向下兼容英语二）。

---

## 2. 考研英语阅读的核心特征

### 2.1 基本参数（英语一/二共性）

| 维度 | 说明 |
| --- | --- |
| CEFR 对应 | B2–C1（英语二偏 B2，英语一偏 C1，统一按 B2+ 处理） |
| 大纲词汇量 | 约 5,500 词（英语一允许 3% 超纲词，英语二未明确；统一按"可能遇到少量超纲词"处理） |
| 语法地位 | 非显性考点，但长难句理解是事实上的核心门槛 |
| 句法复杂度 | 中高至高：从句嵌套、插入语、长距离修饰是常态 |
| 阅读理解题型 | 主旨、推断、态度、例证、细节 + 新题型（排序/匹配/小标题） |
| 文本来源 | 英美报刊（The Economist, NYT, The Guardian 等）、经济管理和社科类报刊 |
| 翻译题 | 英译汉（考句法分析和整体理解能力） |

### 2.2 考研英语的核心考察逻辑

考研英语与其他考试的核心区别在于两点，这两点对英语一和英语二都成立：

**第一：长难句是核心门槛**

考研英语的文章来自英美报刊，句子复杂度显著高于高考和四六级。典型特征：

- 多层从句嵌套（定语从句内再嵌状语从句）
- 长距离主谓分离（主语和谓语之间插入大量修饰成分）
- 插入语和同位语打断句子流
- 复杂并列结构（多个并列成分长度不等）
- 名词化结构导致信息压缩

英语一的长难句密度和复杂度高于英语二，但两者的核心障碍类型一致。

**第二：深度推理是核心区分度**

考研的题目不只要求找到信息，还要求理解：

- 作者的隐含态度（支持/反对/警告/局限性认同）
- 论证逻辑（例子服务于哪个观点）
- 段落间的逻辑关系（转折、递进、因果）
- 词汇的隐含情感色彩

英语一的推理深度高于英语二，但"需要推理"这个特征是考研与高考/四六级的本质区别。

## 3. 各组件在 `kaoyan` variant 下的解析策略

### 3.1 `VocabHighlight` · 考试词汇高亮

**解析大方向**：侧重**熟词僻义和词义辨析**，而不是基础词义覆盖。考研用户的词汇量已经较大（5,500+），核心障碍不是"不认识"，而是"认识但不知道这里什么意思"。

| 维度 | kaoyan variant 策略 |
| --- | --- |
| **选词优先级** | ① 熟词僻义（如 address=解决, fashion=方式, issue=发行）② 考研高频难词（如 scrutiny, contention, consensus）③ 词根词缀可推断的超纲词 |
| **`exam_tags`** | exam_tags 不由 LLM 输出，而是由 PostgreSQL 词典数据库自动匹配。LLM 只需尽量标注考试相关词汇，用户点击查词时，单词卡片会自动显示数据库中已标注的 exam_tag（如 `Kaoyan`、`CET-6`）。因此 prompt 中无需要求 LLM 输出 exam_tags，只需引导其优先选择考研相关词汇即可 |
| **密度控制** | 中等（5–8 个/篇）——聚焦高价值词，不要标基础词。annotation_density 控制每句标注上限，以避免前端显示杂乱为主，应适当放宽；总量偏好通过 prompt 策略引导。标注质量优先于数量，不适配的不硬标 |
| **不应标注** | CET-4 级别的常见词汇、已是常见义的普通词 |

**与高考 variant 的关键区别**：

高考标注的是"用户不认识的考试词"，考研标注的是"用户认识但这里用了不常见的意思"。

### 3.2 `PhraseGloss` · 短语/搭配释义

**解析大方向**：侧重**语篇连接短语和论证性表达**，而不只是固定搭配。

| 维度 | kaoyan variant 策略 |
| --- | --- |
| **标注优先级** | ① 语篇连接词组（in that, insofar as, by virtue of, notwithstanding）② 论证性表达（on the grounds that, to the extent that）③ 报刊中的习惯用法（at stake, in the wake of, fall short of） |
| **`phrase_type`** | 以 `collocation` 为主，较多 `idiom` |
| **`zh` 释义风格** | 给出中文意思 + 简要说明其在论证中的功能（如"in that = 因为，用于引出具体原因"） |
| **密度控制** | 中等（4–7 个/篇） |

**为什么考研场景下语篇连接短语特别重要**：

考研英语的文章来自英美报刊，大量使用复杂的语篇连接短语来构建论证结构。理解这些短语的逻辑功能（因果、转折、让步、限定），是考研阅读推理题的关键。

### 3.3 `ContextGloss` · 语境特殊义

**解析大方向**：考研文本中**熟词僻义和隐含情感色彩**显著多于高考和四六级，ContextGloss 的价值显著提升。

| 维度 | kaoyan variant 策略 |
| --- | --- |
| **标注场景** | ① 熟词僻义（如 novel=新奇的, harbor=怀有, coin=创造）② 报刊体中的隐喻表达（如 navigate the political landscape）③ 带有情感色彩的词（如 alarming, modest, mere 的隐含态度） |
| **`gloss`** | 给出该词在此处的精确含义 |
| **`reason`** | 应特别指出为什么常见义在这里不对，并提示"考研中常通过熟词僻义考察精确理解" |
| **密度控制** | 中等（2–3 个/篇） |

### 3.4 `GrammarNote` · 语法旁注

**解析大方向**：与高考的"显性语法教学"不同，考研场景下 GrammarNote 的目标是**帮助用户拆解长难句结构**，而不是教语法规则。语法在考研中是"理解的工具"而非"知识点"。

| 维度 | kaoyan variant 策略 |
| --- | --- |
| **讲解方式** | 结构分析为主："这里的从句修饰谁、插入语可以暂时跳过、主语和谓语被拆开了" |
| **术语使用** | 可使用标准语法术语（考研用户有足够的语法知识基础） |
| **`note_zh` 风格** | 应包含：① 这个结构在句中起什么作用 ② 它修饰/限定的是什么 ③ 对理解句意的关键提示 |
| **密度控制** | 偏高（4–6 个/篇）——考研文本复杂结构密集。但标注质量优先于数量，如果文本较简单，标注数量自然减少，不要为了凑数而硬标不适配的结构 |

#### 考研重点语法结构清单

与高考不同，考研的语法重点不是"考什么规则"，而是"什么结构会阻碍理解"。LLM 不需要输出固定的标签格式，但应按照考研语法分析的风格和术语来命名 `label`、撰写 `note_zh`。以下语法结构按优先级排列：

### 第一梯队：考研阅读核心障碍结构（文本中存在则标）

| 结构类型 | 为什么是考研核心障碍 | `note_zh` 应如何写 |
| --- | --- | --- |
| **多层从句嵌套** | 考研文本常出现 2–3 层嵌套，读者极易丢失主干 | 明确指出"这个从句修饰谁"，帮用户回到主干 |
| **长距离主谓分离** | 主语和谓语之间插入大量修饰成分，导致主谓关系不清 | 指出"主语和谓语被拆开了，中间的 XX 是插入语/同位语，可先跳过" |
| **复杂并列结构** | 多个并列成分长度不等，难以判断并列边界 | 明确标出"哪些成分是并列的" |
| **非谓语动词作状语/定语** | 分词结构经常与主句主语距离较远，逻辑关系不明显 | 指出"这个分词结构修饰谁、是主动还是被动关系" |

### 第二梯队：常见考研结构（有则标）

- 强调句 `it is...that`
- 倒装句（尤其是部分倒装：only when..., not until..., hardly...when...）
- 虚拟语气（尤其是含蓄虚拟：were it not for..., should this happen...）
- `the more...the more` 等比较结构
- 状语从句（让步、条件、结果）

#### `note_zh` 的写作原则

考研 variant 下的 `note_zh` 应采用"**结构分析"**语气，与高考的"显性教学"和 daily_reading 的"理解辅助"都不同：

|  | daily_reading | gaokao | kaoyan |
| --- | --- | --- | --- |
| 示例 | "has scored 强调评分到目前仍有效" | "这里用现在完成时。高考常考现在完成时与一般过去时的区别" | "这里的非限制性定语从句修饰整个主句，而不是某个名词。理解时可先跳过这个从句，把握主句主干后再回来读" |
| 核心差异 | "这句怎么理解" | "这是什么 + 怎么考" | "这个结构在句中的作用 + 怎么拆解" |

### 3.5 `SentenceAnalysis` · 长难句拆解

这是考研 variant 下**最重要的组件**，因为长难句是考研英语阅读的核心门槛。

**解析大方向**：系统性地拆解复杂句子的层次结构，帮用户掌握"找主干、去枝叶、理层次"的拆句方法。

| 维度 | kaoyan variant 策略 |
| --- | --- |
| **触发条件** | 句子包含从句嵌套 + 修饰成分、或含插入语/同位语/长距离修饰、或总长度超过 25 词 |
| **拆解目标** | "分清层次"——主干是什么、修饰成分各自修饰谁、逻辑关系是什么 |
| **`label`** | 应体现句子的结构特征，如"主谓分离 + 非限定从句补充说明 + 分词状语" |
| **`teach` 风格** | 先抽出主干，再逐层展开修饰成分，最后可说明"这种结构在考研翻译题中常见" |
| **`chunks` 粒度** | **中细粒度**：主语 / 谓语 / 宾语 / 各类从句 / 插入语 / 同位语 / 分词状语。对简单结构可适当合并，复杂结构应拆细 |
| **密度控制** | **偏高**（3–4 句/篇）——考研文本中长难句是核心障碍 |

#### `teach` 示例

> "主干：The study suggests that...（这项研究表明……）。that 引导宾语从句。从句内部，the approach, which was initially designed for... 中的 which 引导非限制性定语从句，修饰 approach，可先跳过。从句主干是 the approach has failed to address....。翻译时建议先译主句，再补充定语从句信息。这种"主谓分离 + 内嵌从句"结构是考研翻译题的高频出题点。"
>

### 3.6 `SentenceTranslation` · 逐句翻译

**解析大方向**：考研用户的翻译需求介于高考和日常阅读之间——用户能理解部分内容，但在长难句上需要翻译确认。同时考研有翻译题，翻译本身就是备考内容。

| 维度 | kaoyan variant 策略 |
| --- | --- |
| **翻译定位** | 理解确认 + 翻译训练 |
| **翻译风格** | 在准确基础上保持自然，对复杂句应体现清晰的句法映射关系 |
| **翻译粒度** | 逐句翻译，长难句应保持完整翻译（不拆分），体现句法关系 |
| **补充说明** | 对特别复杂的句子，可在翻译后用括号补充"此处 XX 指的是…" |

## 4. 整体密度控制与优先级

### 4.1 各组件密度建议

| 组件 | 每篇数量 | 优先级 | 说明 |
| --- | --- | --- | --- |
| `VocabHighlight` | 5–8 | ★★★ | 聚焦熟词僻义和考研高频难词 |
| `PhraseGloss` | 4–7 | ★★★★ | 语篇连接短语和论证性表达关键 |
| `ContextGloss` | 2–3 | ★★★★ | 熟词僻义是考研的核心词汇考察方式 |
| `GrammarNote` | 4–6 | ★★★★ | 结构分析是理解长难句的关键 |
| `SentenceAnalysis` | 3–4 | ★★★★★ | **最高优先级**：长难句拆解是考研核心需求 |
| `SentenceTranslation` | 全量 | ★★★★ | 翻译既是理解工具又是备考训练 |

### 4.2 优先级排序

> `SentenceAnalysis` ＞ `GrammarNote` = `PhraseGloss` = `ContextGloss` ＞ `SentenceTranslation` ＞ `VocabHighlight`
>

与其他 variant 的对比：

|  | daily_reading | gaokao | **kaoyan** |
| --- | --- | --- | --- |
| 最高 | 语境义词汇 + 翻译 | 语法 + 翻译 | **长难句拆解** |
| 高 | 短语搭配 | 考试词汇 + 搭配 | 结构分析 + 语境义 + 语篇短语 |
| 中 | 语法（FonF） | 句子拆解 | 翻译 + 词汇 |
| 低 | 句子拆解 | 语境义 | — |

## 5. Runtime Prompt 注入建议（`scenario_policy` section）

以下是建议在 `kaoyan` variant 下注入到 runtime prompt 的 `scenario_policy` 内容。这是一个统一的 prompt，不区分英语一和英语二。

```
## 场景策略：考研英语

用户正在备考考研英语。考研英语的核心难点是长难句理解和深度推理。文章来自英美主流报刊，句子复杂度和论证深度显著高于其他国内英语考试。请按以下策略调整标注：

### SentenceAnalysis（最高优先级）
- 这是本场景最重要的组件。对包含从句嵌套、插入语打断主谓关系、或超过 25 词的复杂句，必须提供 SentenceAnalysis。
- teach 应先抽出主干，再逐层展开修饰成分，可提示"这种结构在考研翻译题中常见"。
- chunks 应拆到中细粒度：主语 / 谓语 / 宾语 / 各类从句 / 插入语 / 同位语 / 分词状语。简单结构可适当合并，复杂结构应拆细。
- 每篇 3–4 条。

### GrammarNote（高优先级）
- 目标不是教语法规则，而是帮用户拆解复杂结构。
- 优先标注：多层从句嵌套、长距离主谓分离、复杂并列结构、分词状语的逻辑关系。
- note_zh 应说明"这个结构在句中的作用"和"怎么拆解"，而不是"这是什么语法点"。
- 可使用标准语法术语。
- 每篇 4–6 条。

### VocabHighlight
- 用户词汇量约 5500+。不要标注 CET-4 级别的常见词。
- 优先标注：熟词僻义、考研高频难词、词根词缀可推断的超纲词。
- exam_tags 由词典数据库自动匹配，无需 LLM 输出。只需引导 LLM 优先选择考研相关词汇即可。
- 每篇 5–8 个。

### PhraseGloss
- 优先标注语篇连接短语（in that, insofar as, by virtue of）和报刊习惯用法（at stake, in the wake of）。
- zh 释义应包含短语的逻辑功能说明（如 "in that = 因为，用于引出具体原因"）。
- 每篇 4–7 个。

### ContextGloss
- 考研文本中熟词僻义多，这个组件的使用频率应高于高考和日常阅读场景。
- reason 应指出"常见义在这里为什么不对"，可提示"考研常通过熟词僻义考察精确理解"。
- 每篇 2–3 个。

### SentenceTranslation
- 全量翻译。
- 长难句的翻译应保持完整（不拆句），体现清晰的句法映射关系。
- 翻译既服务于理解，也服务于翻译题备考训练。
```

## 6. 与其他 variant 的关键差异

| 维度 | `daily_reading`（intermediate） | `gaokao` | **`kaoyan`** |
| --- | --- | --- | --- |
| **核心目标** | 理解内容 | 掌握语法规则 | **拆解长难句 + 深度推理** |
| **最高优先级组件** | 语境义词汇 + 翻译 | GrammarNote + 翻译 | **SentenceAnalysis** |
| **GrammarNote 的角色** | 理解辅助（FonF） | 显性语法教学 | **结构分析工具** |
| **VocabHighlight 侧重** | 语境义优先 | 考试高频词 | **熟词僻义 + 考研难词** |
| **PhraseGloss 侧重** | 短语释义 | 固定搭配 + 用法 | **语篇连接 + 论证表达** |
| **SentenceAnalysis 触发门槛** | 很复杂才触发 | 20 词 + 从句 | **25 词 + 嵌套/插入语，积极触发** |
| **翻译风格** | 自然意译 | 直译，贴近原文结构 | **准确 + 句法映射清晰** |
