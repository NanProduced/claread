# Product

## Register

product

## Users

Claread 微信小程序服务正在阅读英文材料的中文用户，包括在校大学生、备考用户、职场学习者和轻量学术阅读者。用户通常在手机上碎片化阅读，但仍希望获得比普通翻译、词典或 AI chat 更稳定、更有秩序的英文理解体验。

小程序端的使用场景更接近随身阅读工具：快速粘贴文章、等待解析、回看历史、查词、生词复习、管理配额和个人资产。屏幕有限，交互必须收敛，不能把 Web 级复杂面板压进移动端。

## Product Purpose

小程序端是 Claread 当前稳定客户端基线，用于承载输入分析、解析阅读页、历史记录、生词本、生词复习、收藏、反馈、每日精读、微信登录和配额等主链路。

它的目标不是展示最多能力，而是在微信小程序限制下保持 Claread 的阅读优先体验：文章先于工具，解释服务理解，生词和批注围绕上下文出现。小程序当前 UI 是一个可运行、可继承的冻结基线，但不是 Web 或未来 App 的体验上限。

## Brand Personality

安静、精确、编辑化。

小程序端应有纸质感、安静、苹果味。它参考 Notion 的克制信息组织、Notebook 的纸面亲和力，以及高级杂志的留白、版心和阅读节奏。界面看起来像一份被编辑整理过的英文读物，而不是一个功能堆满的学习工具。

情绪目标是让用户觉得 Claread 可靠、清爽、有审美判断。用户应能专注读完一段英文，而不是不断被按钮、卡片、弹窗和 AI 功能提示打断。

## Anti-references

- 不做通用 AI chat 工具。
- 不做应试英语 App 的高饱和、强运营、强任务感界面。
- 不做词典 App 式的密集列表和强工具栏。
- 不把阅读页做成按钮和卡片堆叠。
- 不用炫技动效、玻璃拟态、浓重渐变或装饰性背景抢走正文注意力。
- 不把小程序平台降级写成 Claread 全局产品限制。
- 不把小程序冻结 UI 当成 Web 端体验上限。

## Design Principles

1. 阅读先于工具。正文、译文、解释和批注永远比入口、按钮、统计和管理能力更重要。
2. 解释像旁注。词义、语法、句式和批注应像纸面上的注释、便签和荧光笔，不像 dashboard 卡片。
3. 能力渐进出现。输入、查词、收藏、反馈、批量管理等操作只在上下文需要时出现。
4. 手机端保持克制。小程序可以用底部栏、sheet、轻卡片和局部浮层，但不能在首屏暴露复杂工作台。
5. 冻结基线可继承，不可放大成全端规则。小程序 UI 是现有稳定实现，Web 端后续可以更高保真。

## Accessibility & Inclusion

小程序端以清晰可读为最低标准：正文和主要文字应保持足够对比度，弱化文字只用于辅助说明；可点击目标应适合拇指触控；动效只用于状态确认和轻量过渡，不依赖动效传达唯一信息。

阅读页应优先保证长文本可读性，避免过小字号、过密行距和过强色彩。批注颜色必须同时依靠位置、形态或文本上下文表达语义，不能只依赖颜色区分类别。
