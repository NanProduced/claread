# Web Reader 信息架构

本文设计 Claread Web Reader 的信息架构、页面结构、核心交互、快捷键、词典浮层、批注系统和历史回看。

设计原则来自 `docs/product/design-context.md`：Calm / Precise / Editorial。Web Reader 不复刻小程序 ScrollView 单栏布局，而是面向桌面宽屏设计，移动 Web 第二波适配。

## 设计原则

- **桌面优先**：利用宽屏空间做信息分层，不是手机页拉伸。
- **信息分层**：正文沉浸 → 渐进披露 → 按需展开。先展示文章和解释，再展示工具。
- **不被小程序限制反向约束**：小程序 ScrollView 降级不是 Web 上限。
- **阅读主线优先**：复杂能力渐进披露，不在第一屏暴露所有操作。
- **反馈、收藏、生词、批注围绕阅读上下文出现**。

## 页面结构

### 桌面三栏布局

```
┌──────────────────────────────────────────────────────────────────┐
│  顶栏：返回 / 标题 / 阅读模式切换 / 配额指示 / 用户头像           │
├────────────────────────────┬─────────────┬───────────────────────┤
│                            │             │                       │
│    主阅读区                │  上下文面板  │   词典 / 批注侧栏     │
│    (60%)                   │  (25%)      │   (15%)               │
│                            │  (可折叠)    │   (可折叠)            │
│  ┌──────────────────────┐  │             │                       │
│  │  段落文本             │  │  逐句翻译   │  ┌─────────────────┐  │
│  │  inline_marks 渲染    │  │  语法笔记   │  │  词典浮层       │  │
│  │  点击词 → 词典浮层    │  │  句尾入口   │  │  词条详情       │  │
│  │  选区 → 批注操作      │  │  词汇笔记   │  │  例句/短语      │  │
│  └──────────────────────┘  │             │  └─────────────────┘  │
│                            │             │                       │
│  sentence_entries chips    │             │  批注列表             │
│  ↓ 展开详情面板            │             │  高亮 / 笔记 / 筛选   │
│                            │             │                       │
├────────────────────────────┴─────────────┴───────────────────────┤
│  底栏：阅读设置 / 字体大小 / 翻译显示模式 / 反馈                   │
└──────────────────────────────────────────────────────────────────┘
```

### 移动 Web 布局（第二波）

- 单栏：主阅读区全宽
- 上下文面板：底部 sheet 上滑
- 词典：底部 sheet
- 批注：全屏列表页

### 阅读模式

| 模式 | 主阅读区 | 上下文面板 | 词典侧栏 |
|------|---------|-----------|---------|
| 沉浸 (Immersive) | 纯正文，隐藏所有标注 | 隐藏 | 隐藏 |
| 标准 (Standard) | 正文 + inline_marks | 翻译 | 按需弹出 |
| 精读 (Intensive) | 正文 + inline_marks + 句尾入口 | 翻译 + 语法笔记 | 常驻侧栏 |

## 核心交互模型

### 主阅读区交互

| 交互 | 触发 | 行为 |
|------|------|------|
| 点击 inline_mark | 点击高亮词/短语 | 词典浮层弹出（右侧栏或 popover） |
| 点击任意词 | 双击非标注词 | 词典查词浮层 |
| 选区批注 | 鼠标选中文本 → 浮出工具栏 | 高亮 / 笔记 / 查词 |
| 句尾入口 | 点击 sentence_entry chip | 上下文面板展开对应详情 |
| 段落悬停 | 鼠标悬停段落 | 显示段落级操作（翻译、笔记） |

### 上下文面板交互

| 交互 | 触发 | 行为 |
|------|------|------|
| 翻译切换 | 快捷键 `T` 或面板按钮 | 逐句翻译 显示/隐藏 |
| 句子聚焦 | 点击主区句子 | 上下文面板滚动到对应翻译/笔记 |
| 语法笔记展开 | 点击 grammar_note | 展开完整语法说明 |
| 词汇笔记展开 | 点击 vocab_highlight | 展开词汇释义 + 语境说明 |

### 词典浮层交互

| 交互 | 触发 | 行为 |
|------|------|------|
| 查词 | 点击 inline_mark 或双击词 | 浮层展示词典结果 |
| 歧义选择 | 返回 `disambiguation` | 候选列表 → 点击选择 → 加载词条 |
| 未找到 | 返回 `not_found` | 空态提示 + 建议反馈 |
| 加入生词本 | 浮层内按钮 | `POST /vocabulary`（第二波） |
| 查看完整词条 | 浮层内链接 | `GET /dict/entry` → 完整释义页 |

## 词典浮层设计

### 触发方式

1. **点击 inline_mark**：直接查 `lookup_text`，`lookup_kind` 决定查词/查短语
2. **双击任意词**：用选中文本调用 `/dict?q=word`
3. **选区后点查词**：用选中文本调用 `/dict?q=text&type=phrase`

### 展示位置

- **桌面**：右侧栏常驻区域，不遮挡正文
- **移动**：底部 sheet

### 内容结构

```
┌─────────────────────────┐
│  word  /wɜːrd/  n.      │  ← display_word + phonetic + pos
│                          │
│  释义                     │
│  1. 单词，字              │  ← definitions[0]
│  2. 消息，话语            │  ← definitions[1]
│                          │
│  短语                     │
│  • in a word 总之        │  ← phrases
│  • word for word 逐字地  │
│                          │
│  例句                     │
│  "What's the word..."    │  ← examples
│  "那个词怎么说……"        │
│                          │
│  [加入生词本]  [完整词条]  │  ← 操作按钮
└─────────────────────────┘
```

### 歧义处理

当 `/dict` 返回 `disambiguation` 时：

```
┌─────────────────────────┐
│  "light" 有多个释义       │
│                          │
│  ○ light¹  n. 光，光线   │  ← candidate 1
│  ○ light²  adj. 轻的     │  ← candidate 2
│  ○ light³  v. 点燃       │  ← candidate 3
│                          │
│  请选择要查看的释义        │
└─────────────────────────┘
```

### 空态

当 `/dict` 返回 `not_found` 时：

```
┌─────────────────────────┐
│  "xyz" 未收录             │
│                          │
│  词典暂未收录此词。       │
│  [反馈缺失]               │
└─────────────────────────┘
```

## 批注系统设计

### 数据模型

复用 `user_annotations` API：

| 字段 | 说明 | Web 增强 |
|------|------|---------|
| `annotation_type` | `highlight / note` | Web 完整支持 |
| `anchor_type` | `sentence / paragraph / text_range` | Web 首期用 `text_range`，精确到字符偏移 |
| `selected_text` | 选中文本 | Web 用浏览器 Selection API 获取 |
| `start_offset / end_offset` | 字符偏移 | Web 基于 `render_text` 坐标 |
| `color` | 5 色高亮 | Web 完整支持 |
| `note` | 笔记文本 | Web 支持富文本（后续） |

### 选区操作工具栏

用户选中文本后，浮出工具栏：

```
┌─────────────────────────────────────────┐
│  🟢  🔵  🟣  🟡  🟢  │  📝  │  🔍  │
│   高亮颜色选择        笔记   查词       │
└─────────────────────────────────────────┘
```

操作流程：
1. 选中文本 → 工具栏浮出
2. 点击颜色 → 创建 `annotation_type=highlight`
3. 点击笔记 → 弹出笔记输入框 → 创建 `annotation_type=note`
4. 点击查词 → 词典浮层查选中文本

### 批注列表（右侧栏）

```
┌─────────────────────────┐
│  批注 (12)    [筛选 ▼]   │
│                          │
│  🟢 "significant"        │  ← 高亮批注
│     ¶2 · 句子级           │
│                          │
│  🔵 "in light of"        │
│     ¶3 · 笔记: 注意短语  │  ← 笔记批注
│                          │
│  🟣 "nevertheless"       │
│     ¶5 · 句子级           │
│                          │
│  [显示全部] [仅笔记]      │
└─────────────────────────┘
```

筛选维度：
- 按颜色
- 按类型（高亮/笔记）
- 按段落

点击批注 → 主阅读区滚动到对应位置并高亮。

### 与 render_scene inline_marks 的关系

- `inline_marks` 是后端分析产出的标注（vocab_highlight / phrase_gloss / context_gloss / grammar_note）
- `user_annotations` 是用户手动创建的批注（highlight / note）
- 两者独立存在，UI 上可叠加显示
- inline_marks 不可编辑，user_annotations 可编辑/删除

### 筛选维度

- `reading_goal`：exam / daily_reading / academic
- `source_type`：user_input / daily_article / imported / ocr
- 日期范围：date_from / date_to
- 搜索：标题/原文关键词（需后端增强）

- 点击卡片 → `GET /records/{id}?include_render_scene=true` → 进入 Reader 页
- Reader 页 URL：`/result/{record_id}`

### 与小程序差异

| 维度 | 小程序 | Web |
|------|--------|-----|
| 列表布局 | 纵向列表 | 卡片网格 + 列表切换 |
| 筛选 | 无 | reading_goal / source_type / 日期 |
| 搜索 | 无 | 标题/原文关键词 |
| 详情入口 | `client_record_id` | `cloud_record_id`（URL 友好） |
| 批量操作 | 无 | 后续可加：批量删除/导出 |

## Academic 模式增强

Academic `RenderSceneModel` 包含 Web 可结构化渲染的额外字段：

| 字段 | 小程序处理 | Web 增强 |
|------|-----------|---------|
| `content_summary` | 可能忽略 | 结构化渲染：overview / research_question / methodology / key_findings / limitations |
| `term_note` inline_marks | 降级为普通标注 | 独立渲染：术语卡片，按 term_category 分组 |
| `logic_note` inline_marks | 降级为普通标注 | 独立渲染：逻辑关系标注，按 logic_type 分色 |
| `interpretation_note` sentence_entries | 降级 | 独立面板：解读笔记 |
| `title` | 可能忽略 | 页面标题 + 历史记录标题 |

## 状态管理设计

### 服务端状态（TanStack Query）

| Query Key | 数据 | 缓存策略 |
|-----------|------|---------|
| `["records", { page, goal, type }]` | 历史记录列表 | staleTime: 30s |
| `["record", id]` | 单条记录详情 | staleTime: 5min |
| `["task", id]` | 任务状态 | 轮询期间 staleTime: 0 |
| `["dict", query]` | 词典结果 | staleTime: 1h（与后端 Cache-Control 对齐） |
| `["quota"]` | 用户配额 | staleTime: 1min |
| `["vocabulary", { page }]` | 生词列表 | staleTime: 30s |
| `["annotations", recordId]` | 批注列表 | staleTime: 30s |

### 客户端 UI 状态（Zustand）

| Store | 数据 |
|-------|------|
| `useAuthStore` | 登录态、session_token、用户信息 |
| `useReaderStore` | 阅读模式、字体大小、翻译显示、面板开关 |
| `useLayoutStore` | 侧栏宽度、面板折叠状态 |
| `useAnnotationStore` | 当前选区、批注工具栏状态 |

### 持久化

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| session_token | httpOnly cookie | 安全 |
| 阅读偏好（字体/模式） | localStorage | 跨 session 保持 |
| 面板布局状态 | localStorage | 跨 session 保持 |
| 滚动位置 | sessionStorage | 同 session 内恢复 |
