# TMP Reader 2.0 文字底板 / 批注层 UI 改造跟踪

## 状态

- 类型：TMP
- 日期：2026-05-20
- 适用范围：`apps/web` Reader 页的原文画布内部显示
- 当前负责人：Codex + 用户协同评估

## 文档目的

本文件用于跟踪本轮 Reader 2.0 UI/UX 改造中，**文字底板与批注层** 的评估、决策和落地顺序。

本轮暂不处理：

- Ask Claread 2.0 工作区
- Reader 整页壳层改版
- 左侧词典 rail 的完整重做
- 每日精读页

本轮只处理：

1. 原文 / 译文显示效果
2. 浏览器选区 / Reader 选区 / 选中反馈
3. 机器批注样式
4. 用户资产样式
5. 激活 / 展开 / 回跳等强调反馈
6. Plate / Plate UI 在上述范围内可提供的帮助

## 当前工作目标

在不改动 Reader 2.0 核心对象模型与桥接真相的前提下，把当前“技术上可用”的文字画布，收口成更符合 Claread `lucid / editorial / tactile` 气质的精读表面。

一句话目标：

`让原文、译文、机器解释和用户痕迹在同一张纸面上说同一种视觉语言。`

## 已确认前提

- Plate runtime / selection bridge / jump bridge 已完成第一轮接入
- 当前问题主要在显示语法、层级节奏和状态反馈，不在对象模型缺失
- 本轮优先做 UI/UX 改造，不先重写交互协议

## 评估主线

### A. 文字底板

- 原文排版
- 译文排版
- 原文与译文的主次关系
- 点词与选区对阅读连续性的影响
- 浏览器原生选区与产品内选区反馈的关系

### B. 机器批注

- `vocab_highlight`
- `phrase_gloss`
- `context_gloss`
- `grammar_note`
- `term_note`
- `logic_note`

重点看：

- 默认态是否克制
- 可点性是否可感知
- 展开解释时回指原文是否足够清楚
- 不同类型之间是否真的可区分

### C. 用户资产

- 用户高亮
- 笔记态
- 收藏态
- 重叠 range
- 句边 gutter marker
- 句后 slip

重点看：

- 是否像“我的阅读痕迹”
- 是否与机器批注拉开层次
- 是否保留精确锚点感

### D. 状态反馈

- 瞬时选中
- 已保存标注
- 当前激活解释
- 路由回跳聚焦
- hover / focus / active / expanded

重点看：

- 这些状态是否属于同一套视觉系统
- 是否存在过强、过弱或互相打架的反馈

### E. Plate / Plate UI 可用能力

- readOnly surface
- floating layer
- selection toolbar
- inline mark rendering
- future comment / annotation affordance

重点看：

- 哪些能力可以直接借
- 哪些只能借行为不能借皮肤
- 哪些地方必须坚持 Claread 自定义实现

## 当前评估结论摘要

### 已有优点

- 原文连续 DOM 和选区桥接基础较好
- 机器批注与用户资产已开始分层
- 句后解释回到原文附近，方向正确
- 多句选区、精确 range 和 route focus 技术底座已具备

### 当前主要问题

- 译文层身份偏弱
- 机器批注的视觉家族还不够收口
- 用户高亮与机器词汇高亮材质过近
- 收藏态与笔记态在正文中的记忆点偏弱
- 选中、保存、激活、回跳四类状态尚未统一成一套视觉语法

## 本轮建议落地顺序

1. 文字底板：原文 / 译文 / 选区显示
2. 机器批注：默认态与激活态
3. 用户资产：高亮 / 收藏 / 笔记 / gutter / slip
4. 状态系统：选中 / 激活 / route focus / expanded
5. Plate / Plate UI 对应改造策略
