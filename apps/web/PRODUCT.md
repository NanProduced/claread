# Product

## Register

product

## Users

Claread Web 面向希望在桌面或浏览器里高保真透读英文文章的中文用户。核心用户包括大学生、备考用户、职场人士、研究/行业材料阅读者，以及已经在小程序完成轻量解读、希望在 Web 上复看、整理、导出的人。

Web 用户比小程序用户更可能处在完整阅读和整理场景：他们有更大屏幕、更长会话、更强的批注、搜索、键盘、导出和分享需求。他们不需要一个专业编辑器或资料工作台，而需要一个友好的深度阅读空间，把英文文章中的词、句、段、篇讲清楚。

## Product Purpose

Claread Web 是 Claread 多端产品中的高保真阅读与产物端。它共享通用后端、canonical result 和 render scene，不为 Web 复制一套业务后端。Web 首期先适配当前小程序已有功能，再逐步扩展 Web 独有的高保真 Reader、语法可视化、AI 辅助、分享页和导出能力。

首期 Web 不优先建设完整 landing / 产品介绍页。功能页面优先，包括粘贴即解读入口、Reader、历史、生词本、复习、用户/配额/设置、分享页和后续导出预览。

Web Reader 默认方向是以原文画布为中心的 Canvas Workspace。文章是主角，所有和文本强绑定的解析结果必须先回到正文附近：词汇和短语在原文上高亮，语法说明和句子拆解在相关句子下方展开，用户笔记和高亮挂载在句子锚点上。词典详情属于画布左侧工具层，AI chatbox 属于画布右侧工具层；两者按需展开或钉住，但不得压缩 65-75ch 的正文文本列，也不要把解析内容统一收进右侧列表。宽屏下可以放宽外层内容容器来容纳段落编号、句后卡和工具避让，但核心阅读列必须保持稳定。

## Brand Personality

Lucid, Editorial, Instrumental, Tactile.

Web 端要让 Claread 的品牌调性更完整地出现：阅读镜头、纸面/笔记质感、精密语法结构、清晰可沉淀的精读产物。它应比小程序更有空间、更有层次、更适合分享和导出，但仍保持安静、友好、可读。

## Anti-references

- 不做 Web 版小程序拉伸。
- 不做 SaaS 管理后台式侧边栏 + 卡片网格。
- 不做 Word / WPS 式专业编辑器。
- 不做 NotebookLM 式多 source 资料工作台。
- 不把 AI chat 放在 Reader 中心，也不把右侧区域做成批注仓库。
- 不做学习打卡、排行榜、连续天数、学习小组。
- 不让生词本压过语法、句子和篇章解读。
- 不为了 Awwwards 风格牺牲功能页可读性。

## Design Principles

1. 功能页优先，品牌感内化。Web 首期先把核心功能做好，不靠完整 landing 建立品牌。
2. Reader 是品牌现场。用户应通过文章解析页记住 Claread，而不是通过营销页记住 Claread。
3. 原文画布优先。初始状态让用户读文章；需要时在原文附近显出语法、词汇、句子拆解和用户笔记。
4. 首版 Reader 先对齐小程序 baseline：`grammar_note` 是局部语法说明，`sentence_analysis` 是长难句拆解卡片。Grammar X-Ray 是后续 Web 增强能力，只有在 workflow schema 提供结构化 xray payload 后再进入 UI 实现。
5. 应用壳要轻。导航可用顶栏、窄 rail、command/search 或浮窗，但不能定义品牌，品牌由阅读画布、词典和批注体验定义。
6. 产物要可分享。分享页、长图、PDF 和 Markdown 是 Web 的重要扩展方向，但不阻塞首期 Reader 主链路。

## Accessibility & Inclusion

Web 端需要同时照顾桌面和移动 Web。首期可以先以桌面功能体验为主，但 Reader 的文字宽度、字号、对比度、键盘可达性、焦点状态、触控目标和响应式退化都必须可持续扩展。

语法、词汇和句子标注不能只依赖颜色传达语义；复杂解释需要可折叠、可跳转、可回到原文锚点。动效应服务定位和理解，不应打断长时间阅读。
